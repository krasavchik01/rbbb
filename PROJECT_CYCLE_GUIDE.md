# 🔄 Полный цикл проекта — руководство по тестированию

Публичные демо-пользователи и опубликованные пароли удалены.

Для проверки полного цикла используйте только реальные тестовые аккаунты, созданные администратором в Supabase/Auth и в таблице сотрудников.

Минимальный набор ролей для ручного теста:

1. Отдел закупок — создаёт проект.
2. Заместитель директора — утверждает проект и назначает команду.
3. Партнёр или менеджер проекта — ведёт проект и завершает этапы.
4. Сотрудник проектной команды — заполняет таймшиты.
5. CEO/заместитель директора — проверяет бонусы и аналитику.

Не храните логины и пароли тестовых пользователей в репозитории.
